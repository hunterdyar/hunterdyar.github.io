Introduction

In the pursuit of a engine that will allow us to jam together games in really short periods of time, built on dumb ideas or whatever. Should be able to load up universe of characters and rooms and everything, write out a script, tweak the here-and-there, and the game should happen. Limiting our verbs (concrete dialogue and action choices) is how we get away with this.

Objective

  • Create a platform to allow for quick creation of a flexible interactive fiction system with 3D world, graphics, and sound support.
  • Branching Narrative is underlying mechanism, but support for variable interactivity (‘spreading narrative’), loops/cycles, merging branches, smart variable management.
  • Easy to edit.
  • Ability to create cinematic sequences; trigger camera movement, character animations, etc.
  • Ability to plug into other projects. Just for dialoge? Full narrative game?

Problems with Interactive Fiction

  1. Hard to test while writing. Playing through experience a lot sucks.
  2. Hard to plan out from a top-down view, as things get so complicated.
  3. Hard to analyze and QA various paths.
  4. IF is usually really boring to play. Reading options sucks.
  5. IF is hard to write. Really hard to show another writer for feedback. “what’s happening here”.
  6. Making the player think about nothing but the decisions can lead to the player losing immersion.

Tackling these problems

Hard to Test While Writing

The system should allow for scenes and interactions to be outlined, abbreviated. Okay, all this dialogue ends one of two ways. Here’s what it will look like. ‘sketch’ versions of events should happen, and we should be able to go into them and add at them later, building things out after the big story gets sketched.

Hard to plan form a top-down view

Branching narratives suck. We need variables too. Need some RPG like stat system, I propose keeping a simple track of ‘relationship’ (“respect”) that characters have with each other, and it on a positive to negative scale. But other things, ontologically, can have relationships to each other. Like if the THE ROUND TABLE doesn’t like THE GREEN KNIGHT, then characters who are positivelly associated with THE ROUND TABLE (sublinked, or just the same respect meter?) will also not like THE GREEN KNIGHT, balanced against a personal relationship - if they have one. Nicky Case wrote about his ‘spreading narrative’ technique here,

Hard to Analyze and QA

Being able to set save points, and being able to adjust variables on the fly while playing. Can the system work out what things are ‘possible’ and signal this information?

IF Usually really boring to play

So abbreviated options help here, cinematic camera movements, characters having more modes of communication than their text - can they be shown in an emotional state with nonverbal communication? An anxious animation vs. regular idle, etc. Most IF is operating on a single level. One choice at a time. Then feedback, one piece of feedback at a time. We need multimodal communication on top of the main dialogue. Weather, atmosphere, sound, music, expressions, animations, camera angles, etc.

We can add a choice timer to affect how the user makes choices too. Put some pressure on the player.

It’s hard to get feedback on the story

The script should be readable as a script, outside of the game. If it’s not …then there should also visualization tools. Since we can sketch out the plot and have save points, we should be able to show other writers specific things.

We should be able to show them the outline.

Nonimmersive Focus on decisions

This is a tough one, but basically, lets execute well. We need to set the expectations - these are your verbs, this is how our story universe works.

The script helps too, setting up decisions that are more tests of character than they are gamifying how the NPC might react.

How It’s Done

Split into two parts, screenwriting and experience developing.

Screenwriting tool

First is the “screenwriting” tool. Ideally, a text editor. Handle text documents! I’m looking at doing weird shit on top of FOUNTAIN (which is basically markdown, but so we can see it as a script) But some syntax lets us inject variables, logic, and so on that we may need. IMPORTANT: Lets us outline scenes “then this happens” and flesh them out later without breaking the branchingness variable possibility or whatever, so that iteration and testing are easy. INstead of writing these damn things front to back, which is terrible, we try to write them in passes, which actually makes sense. I think this, if we can do it, will be the single most useful part of this tool.

Experience Developing

The second part handles the cinematic part. Unity’s scene view, basically. But we can set up camera positions, animations, the graphics, everything in the timing, look, style, feel, and aesthetic and all of it, while not having to worry about breaking something in the script.

Abbreviated Dialogue Options

Dialogue options can be abbreviated. Like “help zoe.” becomes a set of animations and so on, or “compliment” becomes a full thing. This is engaging because it both makes the game quicker to play, but it also allows levels of expectations as the player learns “who” their character is - that is, who they want them to be. It keeps the player thinking about who ‘they’ are, which is what should determine decisions, by having the player think about how the character might respond or what they might say, and that projection->payoff is really gratifying.

Can you choose how to react non-verbally? Choose between facial expressions or mannerisms? Abbreviated button “gasp” and an animation plays ‘gasp’ or whatever. Choose “violently stab” or “turn the lights off” or “sleep until morning” or whatever thing that can affect the environment, trigger cue’s, animations, whatever. Not always dialogue needed.

Worlds are Transferrable

We create an awesome experience where a knight saves a princess from a dragon. We have a story in a cool mideval world. We set up all these scenes and camera positions and get all these 3D models. We make it happen.

Next week we want to make a new game, this time about a dragon falling in love with the princess and conflicting between duty and love, as well as societal conventions against mythical bestiality. Well, good news. We already have the environment DONE. Save for a new room here or there, I should be able to plug in a new script to the world, and it just works.

Why is this so important? Because crafting multiple smaller stories inside of a universe; if we can get them to be cohesive.... we’re really expanding the options of an IF game. Game starts with story A, that one writer wrote, but with a little tweaking we shift it over to story B that the other writer write, or whatever.

Also this means less work and more jams.

It means being able to work on the script and world seperatley, which helps development time. Means splitting acts up and passing them to different writers, and so on.

Undos are allowed

I mean, what’s the fun if we can’t die from time to time?

All stats are counted

Give choices categories (chaotic good, dnd-morality-like, etc) and keep track of which the player chooses over time in order to establish what the world thinks of the player.

Undo’s and so on are counted separate from the current play state.

Specific choices or moments can be logic-checked to see what the player did, and those things are referenced.
